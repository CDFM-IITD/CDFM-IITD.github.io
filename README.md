## Repository Structure


## Adding a New Structure (AI4Mat)

<details>
<summary>Click to expand instructions</summary>

### Step 1 — Create the material page

Copy `_ai4mat/template.md` and rename it using the material's folder name (e.g. `_ai4mat/perovskite-abx3.md`).

Edit the front matter:

```yaml
---
published: true
title: "Perovskite ABX₃"       # display name (subscript via Unicode or HTML)
formula: "ABX<sub>3</sub>"     # shown in the card (HTML ok)
has_dos: true                   # set false if not available
has_band: true
has_aimd: true
---
```

Fill in the page body with the structure viewer, DOS/band plots, and AIMD viewer. Use `_ai4mat/1D-Cu4I8_Hdabco_4.md` as a working reference.

### Step 2 — Add assets

Create the folder `assets/ai4mat/<material-name>/` and put files there:

| File | Used for |
|---|---|
| `structure.vasp` | 3D structure viewer (3Dmol.js reads VASP POSCAR format) |
| `dos.png` | DOS plot (static image) |
| `band.png` | Band structure plot (static image) |
| `trajectory.json` | AIMD trajectory (generated by the parse script below) |

### Step 3 — Generate `trajectory.json` from XDATCAR

Use the script `/tmp/parse_trajectory.py` (or copy it and update the three paths at the top):

```python
XDATCAR = '/path/to/XDATCAR'
POSCAR  = '/path/to/structure.vasp'
OUTPUT  = 'assets/ai4mat/<material-name>/trajectory.json'
```

Run with `python3 parse_trajectory.py`. The script unwraps atomic positions using the minimum-image convention and outputs a flat JSON array format compatible with the Three.js AIMD viewer.

### Step 4 — Commit and push

```bash
git add _ai4mat/<material-name>.md assets/ai4mat/<material-name>/
git commit -m "Add <material-name> material page"
git push
```

The new structure appears automatically on the [/ai4mat/](https://cdfm-iitd.github.io/ai4mat/) listing page.

</details>
